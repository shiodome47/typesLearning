TypeScript オブジェクトの基本

構造をガッチリ定義する：
オブジェクト型

JavaScriptのオブジェクト（{}）は何でも自由に入れられますが、TypeScriptでは「どんな名前のプロパティが、何型で入っているべきか」という設計図（型枠）を厳密に定義します。

1. 変数へのオブジェクト代入

型アノテーション (期待する構造)
const user: {
name: string
name: "Alice"
age: number
age: 25
isAdmin: boolean
isAdmin: false
} = ...;
Type error message here.
実際の値を代入する

2. 関数引数のオブジェクトと任意（?）

関数を呼び出す
function getLabel
引数 item の型ゲート
id: number
必須
label: string
必須
count?: number
任意 (?)
return "id:" + item.id + " label:" + item.label;
戻り値 (string)
---

引数として渡すオブジェクトの型の中に count?: number と書かれています。
この ? (オプショナルプロパティ) があるおかげで、count は「あってもなくても良い」という柔軟なルールになります。APIのオプション設定などを渡す際によく使われるテクニックです。
