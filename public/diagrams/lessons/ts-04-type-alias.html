TypeScript コード整理術

型に名前をつけて使い回す：
型エイリアス (Type Alias)

type 名前 = { ... } と定義することで、オブジェクトの構造を「共通の設計図」として登録できます。何度も同じ型を書く手間が省け、コードが圧倒的に読みやすくなります。

共通の設計図
type User = {
  name: string;
  age: number;
  isAdmin: boolean;
};
変数 (データの作成)
同じ設計図（User）を使って作成
const alice: User = {
  name: "Alice",
  age: 25,
  isAdmin: false,
};
関数 (データの利用)
同じ設計図（User）を要求する
user: User
function greetUser(...) {
  return "Hello, " + user.name +
         "! Age: " + user.age;
}
alice
{ name: "Alice", age: 25... }
> console.log(greetUser(alice));

2. Type Alias（型エイリアス）のメリット

もし使わなかったら？（読みにくい）
function greetUser(user: { name: string; age: number; isAdmin: boolean }): string {
  ...
}

引数の部分に直接型を書くと、コードが横に長くなり、非常に読みにくくなります。また、他の関数でも同じ型を使いたい場合、何度もコピペする必要があり、変更があった時に修正漏れが起きやすくなります。

使った場合（スッキリ・安全）
type User = { ... };

function greetUser(user: User): string {
  ...
}

User という一言で済むようになり、関数の目的がハッキリします。また、変数側と関数側が「同じ設計図を参照している」ことが保証されるため、タイポなどのミスをコンパイラが確実に防いでくれます。
