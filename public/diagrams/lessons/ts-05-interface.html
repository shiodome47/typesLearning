TypeScript 型の拡張と互換性

基礎を引き継ぎ、拡張する：
interface と extends

extends を使うと、既存の型のプロパティを丸ごと引き継いで（コピーして）、新しいプロパティを追加した「拡張版の型」を簡単に作ることができます。
さらに、関数に渡すときは「必要なプロパティさえ持っていれば、余分なものがあっても受け入れる」という柔軟な性質を持ちます。

1. extends による「プロパティの引き継ぎ」

ベースの型 (基本セット)
interface Animal {
name: string
sound: string
}
name:string
sound:string
extends
拡張された型 (追加セット)
interface Dog extends Animal {
name: string
sound: string
追加
breed: string
}

2. 関数に渡すときの「型の互換性」チェック

Dog 型のデータを関数に渡す
const dog: Dog = {
  name: "Pochi",
  sound: "Woof",
  breed: "Shiba",
};
dog (Dog型)
name: "Pochi"
sound: "Woof"
breed: "Shiba"
function describeAnimal
引数 animal の要求ゲート Animal 型
name (必須)
sound (必須)
breed (要求していないので無視)
return animal.name + " says " + animal.sound;
戻り値 (string)
---

TypeScriptの「構造的型付け (Duck Typing)」

関数 describeAnimal は Animal 型を要求していますが、そこに Dog 型を渡してもエラーになりません。
なぜなら、TypeScriptは「名前がAnimalかどうか」ではなく、「中身がAnimalに必要なプロパティ（nameとsound）を全て持っているか？」という構造だけをチェックするからです。breed という余分なプロパティが含まれていても、「必要なものは全部あるからヨシ！」として安全に受け入れてくれます。
