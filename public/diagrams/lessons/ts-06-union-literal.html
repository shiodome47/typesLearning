TypeScript 型の絞り込み

特定の値だけを許可する：
ユニオン型 と リテラル型

"文字列" そのものを型として扱い（リテラル型）、それを |（ユニオン型）で繋ぐことで、「絶対にこの中から選ばなければならない専用ゲート」を作ることができます。

1. 型ゲートと安全な振り分けシミュレーション

関数に文字列を渡す
ユニオン型 (Union Types)
type Status = "loading" | "success" | "error";
引数チェックゲート
function showMessage(status: Status)
Argument of type '"unknown"' is not assignable to parameter of type 'Status'.
switch (status) {
case "loading":
return "読み込み中...";
case "success":
return "完了しました";
case "error":
return "エラーが発生しました";
}
戻り値 (string)
---

2. なぜこの書き方が安全なのか？

string 型の場合（危険）
function show(status: string) {
  ...
}
// タイポしても気付けない！
show("succes"); // バグの温床

引数を単なる string にしてしまうと、どんな文字列でも関数に渡せてしまいます。もしスペルミスをしてもエラーにならないため、switch文のどのケースにも当てはまらず、原因不明のバグを生み出してしまいます。

ユニオン＋リテラル型の場合（安全）
function show(status: Status) {
  ...
}
// 実行前にエディタが赤線で教えてくれる！
show("succes");

特定のリテラル（文字そのもの）を | で繋いだ型を指定することで、TypeScriptは「この3つ以外の文字列は絶対に許さない」という強力なバリデーションを行ってくれます。エディタの入力補完（サジェスト）も効くため、開発体験も劇的に向上します。
