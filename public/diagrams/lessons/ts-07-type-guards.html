TypeScript 型の絞り込み

曖昧な型を確定させる：
型ガード (Type Guard)

複数の型が混ざった「ユニオン型（A | B）」を受け取ったとき、そのままでは安全に処理できません。
JavaScriptの判定機能（typeof や in）を使って条件分岐することで、TypeScriptに「このルートを通ったなら型はAで確定だ！」と教える技術を型ガードと呼びます。

1. typeof を使った基本的な型ガード

引数 (string | number)
function formatValue(value: string | number) {
if (typeof value === "string") {
条件クリア：value は string に確定！
return "文字列: " + value;
}
条件外：消去法で value は number に確定！
return "数値: " + value;
}
戻り値 (string)
---

2. in 演算子を使った型ガード

引数 (Cat | Dog)
function speakPet(pet: Cat | Dog) {
if ("meow" in pet) {
meowプロパティを発見！ pet は Cat に確定！
pet.meow(); // 補完が効く
} else {
meowが無いなら、pet は Dog に確定！
pet.bark(); // 補完が効く
}
Console
>

"プロパティ名" in オブジェクト は、オブジェクトがその名前のプロパティを持っているかを調べるJavaScriptの機能です。
TypeScriptはこれを賢く解釈し、「meowを持っているなら、ユニオン型（Cat | Dog）のうちCatしかありえない！」と自動的に型を絞り込んでくれます。

3. カスタム型ガード（"is" キーワード）

判定関数に渡す
型判定専門の検査所
function isCat(pet: Pet): pet is Cat {
  
return pet.kind === "cat";

}
戻り値 (boolean)
---
コンパイラへの伝達
---

戻り値の型に pet is Cat と書くことで、「この関数が true を返したら、渡された pet は間違いなく Cat 型だとして扱ってよい」という「お墨付き（保証書）」をTypeScriptに与えることができます。
複雑な判定ロジックを別の関数に切り出したい時に非常に便利です。
