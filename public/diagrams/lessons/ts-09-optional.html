安全なプロパティアクセス

エラーで止まらない！
?. と ?? の連携プレー

データが存在しない（null や undefined）かもしれないオブジェクトの奥深くへアクセスする時、
オプショナルチェーン（?.） と Nullish Coalescing（??） を使えば、アプリをクラッシュさせずに安全に「代わりの値」を取り出せます。

1. 深い階層への安全なアクセス

function getCity(user: User) {
  return user?.address?.city ?? "不明";
}
関数を呼び出す
プロパティ確認
?.address
プロパティ確認
スキップ (短絡評価)
?.city
null/undefined判定
?? "不明"
戻り値
---

2. 引数自体が null の場合の安全なアクセス

function getName(u: User | null) {
  return u?.name ?? "ゲスト";
}
関数を呼び出す
プロパティ確認
?.name
null/undefined判定
?? "ゲスト"
戻り値
---
なぜ || ではなく ?? を使うの？
||（OR演算子）は、値が空文字（""）や 0、false の場合でも「偽」と判定して右側の値を使ってしまいます。
一方 ??（Nullish Coalescing）は、本当にデータが無い（null か undefined）時にだけ右側の予備の値を使うため、より安全で正確なフォールバック（代替）が可能です。
