元データを壊さず新しく作る：
イミュータブル (不変) な配列操作

push() や splice() のように元の配列を直接書き換える操作（破壊的変更）は、バグの温床になります。
常に「元の配列をコピーして、新しい配列を作り直す」という安全なアプローチを視覚的に体感してください。

Todo アプリ・シミュレーター

Step 1 現在の配列 (todos) ※これは書き換えずに残します
処理中...
Step 2 新しく生成された配列 (return される値)
ここに新しい配列が作られます
追加 (スプレッド構文)
function addTodo(todos, title) {
  const newTodo = { id: ..., title, ... };
  return [...todos, newTodo];
}

...todos で現在の要素をすべて「新しい箱（配列）」に展開してコピーし、最後に新しい要素を付け足して返します。

更新 (map メソッド)
function toggle(todos, id) {
  return todos.map(t =>
    t.id === id
      ? { ...t, done: !t.done }
      : t
  );
}

対象のIDを見つけたら、オブジェクトの中身も ...t でコピーし、done だけを反転させた「新しいオブジェクト」にすり替えます。

削除 (filter メソッド)
function delete(todos, id) {
  return todos.filter(t =>
    t.id !== id
  );
}

指定されたIDと「一致しない（!==）」ものだけを通過させ、対象の要素が除外された新しい配列を作ります。
