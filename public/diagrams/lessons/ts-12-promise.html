TypeScript / JavaScript 非同期処理

「後で結果を返す」という約束：
Promise の図解

時間がかかる処理（通信やタイマーなど）を行うとき、関数は結果の代わりに「Promise（引換券）」を即座に返します。
処理が終わった後、成功なら .then() へ、失敗なら .catch() へと自動的に結果が運ばれる仕組みを見てみましょう。

非同期関数を呼び出す
fetchUser(userId)
  .then((user) => {
    console.log(user.name);
  })
  .catch((error) => {
    console.error(error);
  });

Pending (待機中)

関数を呼び出した直後の初期状態です。非同期処理（通信やタイマーなど）が終わるのを裏側で待っている状態です。この時点では「値」はまだ存在しません。

Fulfilled (成功)

処理が成功し、resolve(データ) が呼ばれた状態です。Promiseの中にデータが格納され、自動的に .then() のブロックへデータが渡されます。

Rejected (失敗)

エラーが発生し、reject(エラー) が呼ばれた状態です。エラー情報が格納され、.then() を飛び越えて .catch() へ直行します。
