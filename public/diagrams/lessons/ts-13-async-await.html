モダンな非同期処理

処理を「一時停止」して待つ：
async/await と try/catch

await を使うと、Promiseの完了を「その行で一時停止して」待つことができます。結果が返ってきたら次の行へ進み、エラー（throw）が起きれば catch ブロックへ一気にジャンプします。同期処理のように上から下へ素直に読めるのが最大のメリットです。

呼び出し側 (main)
async function main() {
try {
const user = await fetchUser(1);
待機中...
console.log(user.name); // 完了後に実行される
}
catch (error) {
console.error(error.message);
}
}

ポイント解説

await は「そこで待つ」

await がついた非同期関数が呼ばれると、裏側で処理が終わるまで その行でプログラムが一時停止 します。そのため、次の行の console.log は、確実にデータ（user）が手に入ってから実行されます。
（※ブラウザ全体の画面がフリーズするわけではありません）

throw と try/catch

非同期関数の中でエラーが発生し throw されると、処理はそこから呼び出し元の catch ブロックへワープします。そのため、try ブロック内の残りの行（console.logなど）は実行されず、安全にエラー処理だけを行うことができます。
