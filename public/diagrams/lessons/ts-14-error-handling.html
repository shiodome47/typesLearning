安全なデータの受け取り方

正体不明のデータを扱う：
unknown と try/catch

外部から来るJSONデータや、キャッチしたエラーオブジェクトは「何が入っているか分からない」ため、TypeScriptでは安全な unknown 型として扱います。
使う前に「型のチェック（instanceof Error など）」を行うことで、安全にプロパティへアクセスできます。

関数を呼び出す
関数の戻り値 (Result)
---
function parseJSON(jsonStr) 戻り値: unknown
try {
const result: unknown = JSON.parse(jsonStr);
console.log("パース成功:", result);
return result;
}
catch (error: unknown) {
if (error instanceof Error) {
console.error("パースエラー:", error.message);
} else {
console.error("Unknown error");
}
return null;
}
Console
>

ポイント解説

any と unknown の違い

any は「型チェックを放棄する」無法地帯です。
一方 unknown は「中身が何かわからない箱」です。箱を開けて中身を確認する（型チェックする）までは、プロパティへのアクセスやメソッドの呼び出しをコンパイラが許してくれません。これにより、実行時エラーを未然に防ぎます。

instanceof による「型ガード」

JavaScriptの catch (error) には、Errorオブジェクト以外にも単なる文字列や数字が投げられる可能性があります（例: throw "エラーです"）。
そのため、if (error instanceof Error) とチェックすることで、TypeScriptは「このブロック内なら確実に error.message が存在する！」と推論（型を絞り込み）してくれます。
