実践的な非同期データ取得

外部のデータに型をつける：
fetch API と 型アサーション

APIから取得したデータは、TypeScriptにとって「中身が何かわからない状態（any または unknown）」です。
HTTPステータスをチェックし、as User を使って「これはUser型だよ」と教え込む（キャストする）ことで、安全にプロパティへアクセスできるようになります。

関数を呼び出す
async function main() {
try {
const user = await fetchUser(1);
console.log(user.name);
console.log(user.email);
} catch (error) {
console.error(error.message);
}
}
function fetchUser(userId) 戻り値: Promise<User>
const response = await fetch(
  ".../users/" + userId
);
if (!response.ok) {
  throw new Error(\`HTTP error: \${response.status}\`);
}
// .json() は unknown (または any) を返す
const data = await response.json()
  as User;
return data;

実務で必須のテクニック解説

as User (型アサーション) は応急処置

response.json() が返す値は、TypeScriptには中身が推論できないため any や unknown になります。
as User を使うと、コンパイラに「私を信じて。これはUser型だ」と強制的に認識させることができます。手軽ですが、実際のデータがUser型と違っていてもエラーにならないため、より堅牢なアプリでは「Zod」などのバリデーションライブラリを使います。

!response.ok のチェックを忘れずに

fetch APIの罠として、「404 (Not Found) や 500 (Internal Server Error) などのHTTPエラーが返ってきても、Promiseは reject（失敗）されない」という仕様があります。
そのため、response.ok (ステータスが200番台か) を自分でチェックし、ダメなら手動で throw new Error() を投げてあげる必要があります。これを忘れると、エラー画面のHTMLをJSONとしてパースしようとしてアプリがクラッシュします。
