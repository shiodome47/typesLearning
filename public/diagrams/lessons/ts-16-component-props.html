安全にデータを受け渡す：
Props と オプショナル型

コンポーネントが受け取るデータ（Props）の形を type で定義することで、呼び出し側のミス（渡し忘れや型の間違い）をコンパイル時に防ぐことができます。
また、? をつけることで、「渡しても渡さなくても良い」柔軟な設計が可能になります。

呼び出し側 (親コンポーネント)
型チェックゲート
type GreetingProps = {
name: string;
age: number;
isAdmin?: boolean;
};
Property 'age' is missing in type '{ name: string; }'
子コンポーネント (Greeting)
function Greeting({ name, age, isAdmin }: GreetingProps) {
// undefined の場合は false 扱いになる
const adminLabel = isAdmin ? " [Admin]" : "";
return (
  <p>
    Hello, {name} ({age}){adminLabel}
  </p>
);
}

React + TS の実務ポイント

boolean型の省略記法

JSXで isAdmin={true} と書く代わりに、単に isAdmin と書くだけで「trueを渡した」ことになります。
逆に、何も渡さなかった場合は undefined として扱われます。三項演算子（isAdmin ? A : B）では undefined は「偽 (false)」と判定されるため、デフォルト値のような安全な動作をします。

age={25} の波括弧はなぜ必要？

name="Alice" のようにダブルクォーテーションで囲むと「文字列 (string)」として渡されます。
しかし、数値や真偽値、配列などを渡したい場合は、JSXの中でJavaScriptの世界を開くための 波括弧 {} が必要になります。もし age="25" と書いてしまうと、TypeScriptが「number型にstring型は入れられないよ！」とエラーを出してくれます。
