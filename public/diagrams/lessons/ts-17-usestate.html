useState の型定義

useState に初期値を渡すと、TypeScriptは自動で型を推論してくれます。しかし、初期値が null や空の配列 [] などの場合、将来何が入るか分からないため、ジェネリクス <T> を使って型を明示する必要があります。

コンポーネントのコード
function UserCard() {
// ① 初期値から number と自動推論される
const [count, setCount] = useState(0);
// ② null では推論できないので、型引数 < > を明示する
const [user, setUser] = useState<User | null>(null);
return (
  <div>
    <p>カウント: {count}</p>
    <button onClick={() => setCount(count + 1)}>
      +1
    </button>

    <p>{user ? user.name : "ユーザー未選択"}</p>
    <button onClick={() => setUser({ id: 1, name: "Alice" })}>
      ユーザーを設定
    </button>
  </div>
);
}

型推論（Type Inference）の力

useState(0) のように、文字列、数値、真偽値（boolean）を初期値として渡す場合は、わざわざ <number> と書く必要はありません。TypeScriptが「初期値が0なら、この箱は一生number型だな」と賢く推論してくれるため、コードがスッキリします。

型引数（Generics）が必要なケース

初期値が null や [] の場合、TypeScriptは「この箱はずっとnull（または空の配列）しか入らないのかな？」と勘違いして推論（never[] など）してしまいます。
そのため、「今はnullだけど、後でUser型が入る予定だよ！」とコンパイラに教えるために、<User | null> と明示的に型を定義する必要があります。
