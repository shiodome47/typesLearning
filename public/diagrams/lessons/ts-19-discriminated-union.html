TypeScriptの魔法：
タグ付きユニオン の図解

このコードは、「形（Shape）」という箱に共通の名札（kind）を付けることで、安全かつ自動的に計算を振り分ける美しい設計です。その仕組みを見てみましょう。

1. 型（Type）の構造

3つの異なる図形が定義されていますが、すべてに共通して kind という「名札」がついています。
これらを |（または）でまとめたのが Shape 型です。

Circle
kind: "circle" ← 共通の名札
radius: number ← 円だけのデータ
Rect
kind: "rect" ← 共通の名札
width: number
height: number
Triangle
kind: "triangle" ← 共通の名札
base: number
height: number

type Shape = Circle | Rect | Triangle;

2. 動く！ getArea関数の仕組み

下のボタンを押して、図形データを getArea 関数に送ってみましょう。
switch(shape.kind) が名札を見て、自動的に正しい計算ルートに振り分けます。

① データを渡す
② 名札(kind)で分岐
TypeScriptが判定
switch
(shape.kind)
③ 専用の計算式を適用 ＆ 面積を返す
case "circle"
return Math.PI * radius ** 2;
case "rect"
return width * height;
case "triangle"
return (base * height) / 2;
console.log( getArea(...) );
---

この書き方のすごいところ

switch文で "circle" に分岐した瞬間、TypeScriptは「ここを通るデータは絶対に radius を持っている円だ！」と確信します。そのため、ここで間違って shape.width を使おうとすると、プログラムを実行する前に赤い線でエラーを教えてくれます。（これを型の絞り込み: Type Narrowing と呼びます）
