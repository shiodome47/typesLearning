TypeScript 型推論の極意

「目印」で型を仕分ける：
タグ付きユニオン

Shape 型のように複数の型が混ざっている場合、共通のプロパティ（kind）を「タグ（目印）」として持たせます。TypeScriptはこのタグを見ることで、「今どの図形を扱っているか」を正確に特定し、安全に計算を行えるようにしてくれます。

関数を呼び出す (引数)
function getArea(shape: Shape)
Shape = Circle | Rect | Triangle
① タグ(kind)で仕分ける
switch (shape.kind)
"..."
現在の TypeScript の認識
shape は Circle | Rect | Triangle のどれか
② 確定した型で安全に計算
case "circle":
  return Math.PI * shape.radius ** 2;
case "rect":
  return shape.width * shape.height;
case "triangle":
  return (shape.base * shape.height) / 2;
③ 計算結果 (戻り値)
---

なぜ「タグ」が必要なの？

タグがないとエラーになる

もし kind が無かった場合、関数に入ってきた shape は「円かもしれないし、四角形かもしれない」という曖昧な状態です。この状態でいきなり shape.radius と書くと、TypeScriptは「もし四角形(Rect)だったら radius なんて無いから危険だよ！」とエラーを出して計算をブロックします。

タグがあれば安全にアクセスできる

すべての図形に共通のプロパティ（今回は文字列の kind）を持たせ、switch などで分岐させます。
すると、case "circle" のブロックの中に入った瞬間、TypeScriptは「ここを通ったということは、絶対にCircle型だ！」と確信（型の絞り込み）し、radius を使った計算を安全に許可してくれます。
