網羅性チェック の図解

assertNever と never 型を組み合わせることで、将来「新しい図形（Ellipseなど）」を追加した時に、計算式の書き忘れ（caseの漏れ）をコンパイル時に見つけることができる、非常に強力な設計です。

1. 新しい型（Ellipse）の追加

既存の図形に加えて、新しく Ellipse（楕円） が追加されました。

Circle
kind: "circle"
radius: number
Rect
kind: "rect"
width: number
height: number
Triangle
kind: "triangle"
base: number
height: number
Ellipse (NEW)
kind: "ellipse"
rx: number
ry: number
type Shape = Circle | Rect | Triangle | Ellipse;

2. 網羅性チェックのシミュレーション

各ボタンを押してデータを送信してください。一番下の「未知の図形（想定外のデータ）」ボタンを押すと、assertNever がどのように機能するか確認できます。

② switch分岐

なぜ assertNever が必要なのか？

もしあなたがプログラムを改修し、Shape 型の中に新しく Ellipse（楕円） を追加したとします。しかし、うっかり getArea 関数の中に case "ellipse": を書き忘れてしまったらどうなるでしょうか？

TypeScriptの switch 文は、すべての処理を終えた後、default に流れ着いたデータの型を never 型（絶対に存在してはいけない型） として推論しようとします。

assertNever(x: never) 関数は、引数に never 型しか受け付けない関数です。
もし case "ellipse" を書き忘れると、楕円データが default まで到達してしまい、「Ellipse型は never型 に代入できません！」という真っ赤なエラーをプログラムを書いている最中（コンパイル時）に出して教えてくれます。 これにより、ケースの追加忘れを100%防ぐことができるのです。
