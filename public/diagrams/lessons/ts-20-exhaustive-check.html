網羅性チェック と assertNever

プログラムは一度作って終わりではありません。「将来、新しい図形が追加された時」に、
開発者がうっかり計算式の追加（case）を忘れてしまっても、TypeScriptがコンパイル時に真っ赤なエラーを出して教えてくれる完璧な防御システムです。

function getArea(shape: Shape): number {
  switch (shape.kind) {
    case "circle":
      return Math.PI * shape.radius ** 2;
    case "rect":
      return shape.width * shape.height;
    case "triangle":
      return (shape.base * shape.height) / 2;
    case "ellipse":
      return Math.PI * shape.rx * shape.ry;
    default:
      // 未処理の型が残っていればここでエラーになる！
      return assertNever(shape);
  }
}

なぜ assertNever を使うの？

default 句と never 型の性質

TypeScriptの switch 文は、全ての case を通り抜けて default に到達した変数の型を、「絶対に存在してはいけない型＝ never 型」として推論します。
もし全てのケース（Circle, Rect, Triangle）が正しく書かれていれば、shape は全て処理されるため、defaultに到達する値は「何もない（never）」になります。

コンパイルエラーで未然に防ぐ

そこに新しい型（Ellipse）が追加されると、処理されなかったEllipse型が default に流れ着きます。
assertNever は意図的に「never型しか受け付けない」ように作られた関数です。そのため、Ellipse型が渡されると「neverじゃない型が来たぞ！」とTypeScriptが即座にコンパイルエラーを出し、ケースの書き忘れを開発中に気づかせてくれるのです。
