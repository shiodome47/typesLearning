TypeScript 型推論の真骨頂

構造から型を導き出す：
keyof & ジェネリクス

keyof でオブジェクトの「キー一覧」を取得し、ジェネリクス <T, K extends keyof T> を組み合わせることで、「存在しないキーへのアクセス」を弾き、「戻り値の型」まで完璧に特定する関数の仕組みを見てみましょう。

1. 型の抽出（型の世界）

ベースの型
type Product = {
name : string;
price : number;
inStock : boolean;
};
キーのユニオン型が抽出されます
対応する値の型が抽出されます

2. 安全な関数 getProp の仕組み

下のボタンを押して、オブジェクトとキーを関数に渡してみてください。ジェネリクスがどのように「型の門番」として働き、戻り値の型を決定するかを確認できます。

const p: Product = { ... }
① getProp(p, キー) を実行
② 型チェックと推論 (コンパイル時)
ジェネリクス（型の変数）
function getProp<
  T,
  K extends keyof T
>(
  obj: T,
  key: K
): T[K] {
  return obj[key];
}
③ 実行結果 ＆ 推論された型
出力
---
Argument of type '"weight"' is not assignable to parameter of type '"name" | "price" | "inStock"'.

K extends keyof T の意味

extends は「〜を満たす（〜の部分集合である）」という意味です。つまり K extends keyof T は、「引数key(K) は、オブジェクト(T) が持っているキー一覧(keyof T) の中に含まれていなければならない！」という厳しいルール（制約）を設定しています。これにより、タイポや存在しないキーの指定を未然に防ぎます。
