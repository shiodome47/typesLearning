React フックの極意

去り際のお片付け：
useEffect のクリーンアップ

useEffect の中で関数を return すると、そのコンポーネントが画面から消える時（アンマウント時）に実行されます。裏で動いているタイマーやイベント監視を止めて、「メモリの無駄遣い（メモリリーク）」を防ぐための重要な仕組みです。

2. コードの構造解説

Timerコンポーネント 継続的な処理
function Timer() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    // ① マウント時に開始
    const id = setInterval(() => {
      setCount((c) => c + 1);
    }, 1000);

    // ② アンマウント時に実行される「お片付け」
     return () => clearInterval(id);   }, []);

  return <div>経過秒数: {count}</div>;
}
useWindowWidth カスタムフック イベント監視
function useWindowWidth() {
  const [width, setWidth] = useState(window.innerWidth);

  useEffect(() => {
    // ① マウント時に監視スタート
    const handler = () => setWidth(window.innerWidth);
    window.addEventListener("resize", handler);

    // ② アンマウント時に監視を解除
     return () => window.removeEventListener("resize", handler);   }, []);

  return width;
}

もし return のお片付けを書き忘れたら？

コンポーネントが画面から消えても、裏側の setInterval や addEventListener は**永遠に動き続けます**。再びコンポーネントを表示すると、新しいタイマーが追加され、裏で2つ、3つとタイマーが重複して暴走します。これがアプリをクラッシュさせる「メモリリーク」の原因です。だからこそ、去り際の return () => ... は必須なのです。
