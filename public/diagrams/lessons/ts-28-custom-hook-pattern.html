useState と useEffect を組み合わせた複雑な処理を、1つの関数 useLocalStorage に閉じ込めました。

2. なぜこの書き方なのか？（解説）

① useState の 遅延初期化 (Lazy Initialization)
const [value, setValue] = useState(() => {
  // localStorageへのアクセスは重い処理！
  const item = localStorage.getItem(key);
  return item ? JSON.parse(item) : initialValue;
});
なぜ関数 () => { ... } を渡すの？
LocalStorageからの読み込みは、通常のメモリ操作より時間がかかります。もし関数で包まずに直接書くと、画面が再レンダリングされるたびに毎回重い読み込み処理が走ってしまいます。 関数を渡すことで、Reactは「最初の1回目（マウント時）だけ」この処理を実行してくれるようになります。
② useEffect による自動保存
useEffect(() => {
  localStorage.setItem(key, JSON.stringify(value));
}, [key, value]); // ← 依存配列
なぜ useEffect を使うの？
「値が変わった時」にだけ処理を走らせたいからです。依存配列（[key, value]）を指定することで、value が更新された直後に自動的に保存処理が走ります。
使う側（Appコンポーネント）は「保存すること」を一切意識せずに、ただ setName を呼ぶだけで良くなります。
