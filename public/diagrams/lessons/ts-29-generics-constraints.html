TypeScript 型の門番

型の条件を縛る：
T extends ... (型の制約)

ジェネリクスの <T> は何でも入る万能の箱ですが、extends を付けることで「ただし〇〇の条件を満たすものに限る！」という関所（門番）を作ることができます。これにより、関数の中で安全にプロパティへアクセスできるようになります。

Part 1: T extends object（オブジェクトに限定する）

関数に引数を渡す
function getKeys
function getKeys<
   T extends object

>(obj: T) {
  return Object.keys(obj);
}
Type 'number' does not satisfy the constraint 'object'.
戻り値 (キーの配列)
---
もし extends object が無かったら、42 を渡してもコンパイルエラーにならず、実行時に予期せぬ挙動（空配列が返る等）を引き起こす危険があります。

Part 2: K extends keyof T（キーの存在保証）

const user = {
  id: 1,
  name: "Alice",
  email: "a@b.com"
};
function getProp
function getProp<
  T extends object,
   K extends keyof T

>(obj: T, key: K): T[K] {
  return obj[key];
}
現在の許可リスト (keyof T)
"id" |
"name" |
"email"
Type '"missing"' is not assignable to type '"id" | "name" | "email"'.
戻り値 ( T[K] )
---
「K は T が持つキーのどれかである」という制約です。この2つの型引数を連携させることで、「存在しないプロパティにアクセスして undefined になるエラー」をタイポした瞬間に防いでくれます。
