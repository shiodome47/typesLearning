TypeScript 型の抽出テクニック

関数の「型」を自由自在に操る：
Parameters, ReturnType, Awaited

他の人が作った関数やライブラリの「引数」や「戻り値」と同じ型を使いたい時、手書きでコピーする必要はありません。
TypeScriptのユーティリティ型（Utility Types）を使えば、関数から直接「型だけを吸い出す」ことができます。

1. 関数から「引数」と「戻り値」の型を吸い出す

対象の関数 (formatUser)
function formatUser(
  id: number, name: string
): { id: number; label: string } {
  return { id, label: \`[\${id}] \${name}\` };
}
引数の型 (タプルとして抽出)
type FormatUserParams = Parameters<typeof formatUser>;
---
戻り値の型
type FormatUserReturn = ReturnType<typeof formatUser>;
---

2. Promise の殻を割って中身を取り出す (Awaited)

async function fetchProduct(id) { ... }

// まず ReturnType で戻り値を抽出
type Result = Promise<{ id: number; name: string }>;
Awaited
Awaited<Result>
---
API通信などの非同期関数（async）の戻り値は必ず Promise に包まれています。Awaited は、そのPromiseの殻をパカッと割って、本当に欲しかった中身の型だけを取り出してくれる便利なユーティリティです。

3. 実践：元の型を完全に引き継ぐラッパー関数

ログ出力などの共通処理を挟む「ラッパー関数」を作る時、Parameters と ReturnType を使うことで、「どんな関数を渡しても、元の引数や戻り値の型をそのまま引き継いでくれる」安全な仕組みが作れます。

高階関数 withLogging
function withLogging<T extends (...args: any[]) => any>(fn: T) {
  return (...args: Parameters<T>): ReturnType<T> => {
    console.log("calling:", fn.name);
    return fn(...args);
  };
}
const loggedFormat =
  withLogging(formatUser);
loggedFormat(1, "Alice");
