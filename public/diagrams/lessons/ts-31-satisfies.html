TypeScript 4.9 の神機能

型のチェックと「リテラル推論」を両立：
satisfies 演算子

オブジェクトを作る時、従来の型注釈（:）を使うとせっかくの「具体的な値の情報（リテラル）」が失われてしまいます。
satisfies を使えば、「ルールを満たしているかチェックしつつ、値の元の形はそのまま残す」といういいとこ取りが可能になります。

A. 型注釈 ( : Type )
const A: Record<string, string> = {
  home: "/"
};
処理フロー
型チェック ＆ 型の変換
Record<string, string>
型エラーで弾かれます！
A.home の型
---
安全だが情報が欠落する。 型チェックはしてくれますが、"/" という具体的な情報が string という大雑把な型に丸め込まれてしまいます。
B. 型アサーション ( as )
const B = {
  home: "/"
} as Record<string, string>;
処理フロー
無検査でラベルを貼る
as Record<string, string>
エラーにならず素通り！
B.home の型
---
非常に危険。 コンパイラを黙らせるため、間違った値（42など）を入れてもエラーが出ません。さらに型も string に丸め込まれてしまいます。
Best Practice ✨
C. satisfies 演算子
const C = {
  home: "/"
} satisfies Record<string, string>;
処理フロー
型チェック (変換はしない)
satisfies Record<...>
型エラーで弾かれます！
C.home の型
---
安全 ＋ リテラル型の維持！ 型が間違っていればエラーを出し、正常なら値の形（"/"）をそのまま保持してくれるため、後の処理での型補完が最強になります。
